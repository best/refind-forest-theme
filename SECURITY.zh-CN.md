# 安全策略

[English](SECURITY.md)

本项目会构建 EFI 启动管理器文件，并可将其部署到系统。路径、身份校验、备份、
签名或固件变量处理中的缺陷可能影响启动链，并可能导致机器无法正常启动。

## 报告漏洞

对于疑似漏洞，请使用 GitHub 私密漏洞报告。打开仓库的 **Security** 标签页，
选择 **Report a vulnerability**，然后通过
<https://github.com/best/refind-forest-theme/security/advisories/new> 提交报告。
对于启动链漏洞，或可能暴露签名材料、修改 ESP、改变 NVRAM 状态的弱点，不要
创建公开 issue。如果无法确定报告是否涉及安全问题，请优先使用私密渠道。

请提供受影响的修订版本、涉及的操作、预期行为、实际行为，以及尽可能小且安全
的复现方式。尽可能使用 QEMU/OVMF 或合成测试夹具。必须对所有附件脱敏：不要
包含私钥、证书、令牌、EFI 变量转储、磁盘标识符、机器快照、私有启动配置或
绝对主目录路径。

## 安全调查

如果隔离镜像足以复现问题，就不要在生产启动路径上复现。不要向未经审查的更改
授予 root 权限。在执行任何经授权且可能写入 EFI 系统分区或 NVRAM 的硬件测试
之前，必须准备一条已经测试可用的固件或操作系统恢复路径，并保留经过验证的备份。

一般使用问题和不敏感的缺陷可以使用公开 issue，但必须先对日志和配置进行脱敏。
